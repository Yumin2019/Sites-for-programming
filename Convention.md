# Convention
- 커밋 메세지에 대한 약속
- 타입, 본문, 꼬리말로 이루어져 있음

## 타입(Type) 
- change: 기존에 있는 기능에서 변경사항이 생긴 경우
- feat: 새로운 기능 추가 
- fix: 버그 픽스 
- docs: 문서 수정
- resource: 리소스 추가(string, image 등)
- style: 세미콜론 수정, 코드 정리 등 스타일에 대한 수정
- refactor: 코드 리펙토링
- test: 테스트 코드 추가
- version: 버전을 올리는 경우  


## 본문 내용(Body)
- 추가적으로 덧붙이고 싶은 내용을 적는다.

## 꼬리말(Footer)
- #PR-120 같은 이슈 링크 처리
